# Contribution Guidelines

We welcome contributions! 

To ensure a smooth process, please follow these guidelines when submitting a pull request:

- **Search first:** Check previous suggestions before submitting yours to avoid duplicates.  
- **One suggestion per PR:** Make an individual pull request for each contribution.  
- **Capitalized titles:** Use proper title capitalization ([guide](http://grammar.yourdictionary.com/capitalization/rules-for-capitalization-in-titles.html)).  
- **Keep descriptions concise:** Write short, clear, and informative descriptions.  
- **End with a period:** All descriptions should end with a full stop.  
- **Alphabetical order:** Order link titles alphabetically within each category.  
- **Check spelling and grammar:** Ensure text is error-free.  
- **Trim whitespace:** Set your editor to remove trailing whitespace automatically.  
- **Descriptive PRs and commits:** Make your pull request title and description as clear and informative as possible.  
- **Category improvements welcome:** Feel free to suggest new categories or improvements to existing ones.  
- **Useful commit titles:** Give your commits meaningful and descriptive titles.
